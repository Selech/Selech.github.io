Answers

When looking up a texel value for a given point the point seldom matches the exact center of a texel. Additionally, the size of a pixel may not be equal to the size of the texel in the texture. If the texel is smaller than a pixel we have minification and if it is bigger we have magnification. The filtering mode determines how the application should look up the texel values. There exists three filtering modes in WebGL: Nearest, linear, and mipmap.

Nearest filtering simply returns the value of the closest texel. This often results in jagged edges but is on the other hand very fast. When having minification the result will look blocky and when having magnification the edges can look jagged and wrong. Motion can often also result in moving artifacts.

Linear filtering uses the value of the four nearest texels and returns the interpolated value between them. This gives smoother lines and reduces aliasing but it can still contain unwanted artifacts. When having minification the result will look smooth but still a bit blocky and when having magnification the edges can still look jagged and wrong.

Mipmap filtering generates multiple textures each with half the resolution of the last. This makes it possible for the application to use the texture where the texel size matches the pixel size the best. This will often give the best results but in return requires additional textures.
